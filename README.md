# Netology Cloud Storage

Проект представляет собой хранилище файлов с использованием Java и Spring Boot. Основная цель проекта — обеспечить безопасное хранение и управление файлами пользователей через веб-интерфейс API. В проекте для хранения файлов выбрана локальная директория, но это может быть и жесткий диск, и облачное хранилище. Информация о пользователях и файлах хранится в базе данных (использована PostgreSQL).

## Описание функций

Основные возможности:
- Регистрация новых пользователей `/cloud/user`
- Авторизация и аутентификация пользователей с использованием JWT-токенов `/cloud/login`, `/cloud/logout`
- Управление файлами: загрузка, скачивание, переименование и удаление файлов `/cloud/file`
- Просмотр списка файлов пользователя `/cloud/list`

## Безопасность

Для защиты приложения используются механизмы Spring Security и JWT-токены. Для каждого зарегистрированного пользователя создается уникальный токен, используемый для проверки подлинности запросов. Аутентифицированные пользователи имеют доступ к своим личным данным и файлам.

## Структура классов

- Controllers: Обрабатывают HTTP-запросы и отвечают за взаимодействие с сервисами.
- Services: Обеспечивают бизнес-логику приложения, взаимодействуют с репозиториями и базой данных.
- Repositories: Предоставляют методы для работы с базой данных.
- Entities: Модели данных, используемые для хранения объектов в базе данных.
- DTO: Классы передачи данных, упрощающие обмен информацией между слоями приложения.
- Filters: Фильтры безопасности для обработки запросов и контроля доступа.

## Архитектурные решения

- Используется паттерн MVC (Model-View-Controller) для разделения ответственности компонентов.
- Реализована поддержка CORS для разрешения междоменных запросов (следующий этап).
- Токены хранятся в базе данных и проверяются перед каждым защищенным запросом.

## Настройка окружения

Для запуска проекта используется контейнер Docker с сервером баз данных PostgreSQL.
1. Убедитесь, что установлен Docker.
2. Выполните команду для запуска PostgreSQL-контейнера.
3. Запустите приложение.

## Тестирование

Код покрыт unit-тестами с использованием mockito.
Проведены интеграционные тесты с использованием testcontainers.

## Требования

- Java 11+.
- Maven.
- PostgreSQL (или другая совместимая база данных).
- Apache Tomcat (для развертывания приложения).
